# GBP Post Automation — Context Brief
> Source: Mike DeLong's shared materials (May 2026)
> Purpose: Background context before designing Skywide integration

---

## What This Workflow Does

An **n8n workflow** that automates writing Google Business Profile (GBP) posts for clients. It does NOT publish directly to GBP — it generates and stores content in Google Sheets, ready for review/publishing.

### Node-by-Node Flow

| # | Node | What It Does |
|---|------|-------------|
| 1 | Manual Trigger | Manually kicked off in n8n (not scheduled yet) |
| 2 | Read Locations | Reads rows from the `Topics` sheet — specifically the `Post Topic` column |
| 3 | Loop Over Topics | Split in Batches — processes one topic at a time |
| 4 | HTTP Request | Pulls the client's sitemap (e.g. `https://www.seobrand.com/page-sitemap.xml`) |
| 5 | JS Code | Parses the XML sitemap → extracts all page URLs |
| 6 | Choose Best Link | OpenAI picks the most relevant internal URL for the topic (avoids city/location pages unless needed) |
| 7 | Edit Fields | Hardcodes brand details: Business Name, Industry, City |
| 8 | Generate Post | OpenAI writes the GBP post (<100 words, friendly/local tone, 2–3 paragraphs, no hashtags/emojis) |
| 9 | Generate Image Prompt | OpenAI writes a 1:1 image generation prompt (clean, modern, no text/logos) |
| 10 | Wait 1s | Buffer before write |
| 11 | Append to Sheet | Writes output to `GBP Posts` tab: Business Name, City, State, Post, Generated Date, Image Prompt, Link URL |

## Client Workbooks

### 1. SEO Brand (Single Location)
**Sheets:** `Topics`, `GBP Posts`

**Topics columns:** `Post Topic`, `Status`, `Generated Date`

**Sample topics:**
- SEO tips for Boca Raton businesses to get more qualified leads
- What local SEO is and why it matters for service businesses in Boca Raton
- How to tell if your website is costing you leads
- PPC vs SEO: which one should you start with for faster growth
- How to improve Google rankings without chasing "quick fixes"

**GBP Posts columns:** `Business Name`, `City`, `State`, `Image Prompt`, `Post`, `Link URL`

**Per-location tab columns:** `Post`, `City`, `State`, `Industry`, `Generated Date`

## Key Observations / Known Gaps

1. **Manual trigger only** — no scheduling, no queue management
2. **No direct GBP publishing** — output is stored in Sheets; publishing is manual
3. **Hardcoded per client** — business name, city, sitemap URL baked into each workflow
4. **Single vs. multi-location handled differently** — SEO Brand = one tab + city column; Suncoast = one tab per location
5. **No deduplication** — no check if a topic was already generated
6. **No Skywide integration** — lives entirely in n8n + Google Sheets with no connection to the platform
